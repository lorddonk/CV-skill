# One-Page Length, Page Balance, And File Naming

Prefer a one-page CV whenever reasonably possible, especially for students, recent graduates, and applicants with limited full-time experience. The goal is not merely to fit one page; it is a focused, readable, balanced one-page CV that remains visually consistent with `CV_Shen.docx`.

## Length Control Priority

Before changing font size or layout, control length through content selection and editing:

1. Remove duplicated or repetitive information.
2. Remove low-relevance content.
3. Shorten less relevant experiences.
4. Reduce less relevant experiences to one or two bullets.
5. Combine overlapping bullets.
6. Rewrite long bullets more concisely.
7. Remove generic duties and weak claims.
8. Retain strongest academic, research, professional, technical, and leadership evidence.
9. Slightly adjust paragraph/section spacing.
10. Only then make minor font-size adjustments.

Do not remove degree information, important professional experience, major research, essential dates, required qualifications, or information needed to understand the student's development. If essential content cannot fit, use two balanced pages and briefly explain why.

Avoid a second page with only a few lines; further optimize content/layout so the CV fits one page cleanly.

## Adaptive Font Size

After approved content is inserted:

- If one page looks balanced: keep template font sizes.
- If underfilled: increase all font sizes consistently by about 0.5 pt, up to 1 pt total.
- If slightly over one page: shorten content and tighten spacing first; only then reduce font size slightly.
- Keep body text readable; do not reduce body text below about 9.5 pt without user approval.
- If clearly two pages are needed: use two balanced pages, no tiny fonts, no orphan headings, no awkward experience splits.

Preserve hierarchy when adjusting sizes: name, headings, entry titles, role/date lines, and body text should all move consistently.

## Page Balance Check

Before export, check:

- Page is appropriately filled, not too empty or overcrowded.
- Font size is comfortable.
- Spacing is consistent.
- Visual hierarchy is preserved.
- Headings stay with first entries.
- No experience is split awkwardly.
- Final page looks intentional.
- The document still resembles `CV_Shen.docx`.

## File Naming

Final DOCX must be named:

`CV-[Student English Name].docx`

Examples:

- `CV-Chen Yiran.docx`
- `CV-Tiancheng Shen.docx`

Use the English name from personal information. Do not use generic names such as `output.docx`, `final_cv.docx`, or `student_cv.docx`. If no English name is provided, ask the user to confirm it before final generation.

For multiple program-specific versions:

`CV-[Student English Name]-[University or Program].docx`
