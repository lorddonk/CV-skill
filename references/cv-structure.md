# CV Structure Standards

Use these standards when structuring an English CV for overseas university applications.

## 1. Personal Information

Start the CV with concise contact information:

- Full name
- Email address
- Phone number, if provided
- Address, if provided
- LinkedIn, GitHub, personal website, or portfolio link, if relevant

Rules:

- In Shen-style DOCX output, center the student's name at the top and center the contact line below it.
- Do not include gender, age, date of birth, marital status, nationality, ID number, political status, height, weight, or photo unless the user explicitly says the application requires it.
- Recommend a professional email address, preferably Gmail or an institutional university email.
- Keep this section clean and compact.

## 2. EDUCATION

Write education in reverse chronological order:

- Master's degree first, if applicable.
- Bachelor's degree after that.
- If currently enrolled, include the expected graduation date, such as `Expected: June 2026`.

Each education entry should include:

- Full university name
- Location
- Degree title, such as `MSc in Computer Science` or `BSc in Economics`
- Major or specialization, such as `Artificial Intelligence & Machine Learning`
- Study period or graduation date
- GPA, if strong or necessary
- Ranking, if impressive
- Academic honors, scholarships, or awards
- Relevant coursework

Shen-style entry format:

```text
University Name                                      Location
Degree Name                                         Date / Expected Graduation
- GPA: x / x
- Relevant Coursework: Course 1, Course 2, Course 3, Course 4
- Honors: ...
```

Sort multiple education entries in reverse chronological order. The most recent or expected graduation date appears first.

Relevant coursework rules:

- Include only courses directly related to the application direction.
- Avoid long lists of general courses.
- Prioritize advanced, high-scoring, or application-relevant courses.
- Good examples: Advanced Algorithms, Data Mining, Econometrics, Machine Learning, Financial Modelling, Research Methods, Statistics, Policy Analysis, Experimental Design.

Academic honors rules:

- Include GPA if above 3.5/4.0 or otherwise competitive in the student's grading system.
- Include scholarships, Dean's List, National Scholarship, academic awards, and notable honors.
- Do not exaggerate or invent academic achievements.

## 3. PROFESSIONAL EXPERIENCE

Use the exact section title `PROFESSIONAL EXPERIENCE` by default.

Include:

- Internships
- Full-time work experience
- Part-time professional roles
- Industry projects
- Consulting projects
- Research assistant or teaching assistant roles, if more professional than academic

Shen-style entry format:

```text
Company / Organization Name                          Location
Position Title                                      Date Range
- Bullet point
- Bullet point
```

Each experience should normally include about 4 bullet points and must not exceed 5.

Sort multiple professional entries in reverse chronological order. Use the end date or most recent date in the range.

## 4. ACADEMIC EXPERIENCE

Use the exact section title `ACADEMIC EXPERIENCE` by default.

Include:

- Research projects
- Academic projects
- Highly relevant course-based projects
- Competitions
- Thesis or dissertation experience
- Independent research
- Lab experience

Shen-style entry format:

```text
Project / Research Title                             Location, if applicable
Role / Position                                     Date Range
- Bullet point
- Bullet point
```

For each entry, capture:

- Research question or project objective
- Methods, models, tools, frameworks, datasets, or analytical approaches
- Student's personal contribution
- Outcome, result, finding, ranking, output, or impact

Each project should normally include about 4 bullet points and must not exceed 5. Use fewer bullets for smaller projects.

Sort multiple academic entries in reverse chronological order. Use the end date, competition date, project completion date, or expected/current status.

## 5. SKILLS & AWARDS

Use the exact section title `SKILLS & AWARDS` by default.

Use compact bullets:

- Technical Skills: Python, SQL, R, MATLAB, SPSS, Stata, Java, C++, HTML/CSS, JavaScript, Tableau, Power BI, Excel, TensorFlow, PyTorch.
- Research & Analytical Skills: quantitative analysis, qualitative research, survey design, data visualization, statistical modelling, financial modelling, machine learning, literature review, experimental design.
- Software & Tools: MS Office Suite, Tableau, Power BI, AutoCAD, Revit.
- Language Skills: IELTS, TOEFL, GRE, GMAT, CET-6, or other language qualifications.
- Certifications: AWS Certified Developer, CFA Level I, FRM, Google Data Analytics Certificate, or field-related certificates.
- Awards: scholarships, competition awards, academic honors, Dean's List, or other relevant achievements.

Rules:

- Include only skills the student actually has.
- Prioritize skills relevant to the target program.
- Avoid overloading this section with unrelated tools or generic soft skills.
